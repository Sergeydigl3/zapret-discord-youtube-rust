# 💻 Zapret-Rust TUI

Plug&Play решение для обхода DPI блокировок

---

## ⚙️ Требования к системе

| Операционная система | Требования к окружению | Используемый бэкенд |
| :--- | :--- | :--- |
| **Linux** | `nftables` (или `iptables`), права `root` (для настройки фаервола и установки службы) | `nftables` (через API) / `iptables` |
| **Windows** | Права Администратора (для установки службы и драйвера WinDivert) | `WinDivert` (автоматически скачивается вместе с бинарниками) |

## Быстрый старт

### 1. Сборка из исходников

Для сборки проекта вам понадобится установленный Rust-инструментарий (`cargo`).

```bash
git clone https://github.com/Sergeydigl3/zapret-discord-youtube-rust.git
cd zapret-discord-youtube-rust

cargo build --release
```

Готовый бинарный файл будет находиться по пути `target/release/zapret-rust`.

---

### 1.1 Готовый файл

Если собирать проект не хочется то стабильные версии находятся во вкладке releases. Их нужно положить в любую директорию и открыть.

---

### 2. Запуск в интерактивном режиме (TUI)

Просто запустите утилиту без аргументов. Она сама проверит права и при необходимости запросит пароль:

```bash
./target/release/zapret-rust #Linux
```
На Windows открыть файл можно мышью

> [!TIP]
> При первом запуске перейдите в меню **📥 Загрузчик** (Downloader) -> **⬇️ Скачать всё (по умолчанию)** (Download All). Программа скачает необходимые бинарные файлы `nfqws`/`winws` и актуальные файлы стратегий. После этого выберите интерфейс и стратегию в главном меню и нажмите **▶️ Запустить** (Run).

## 💻 Использование CLI (Неинтерактивный режим)

Утилиту можно запускать из консоли с передачей параметров (например, для использования в собственных скриптах или автозапуске).

### Доступные флаги и опции:

```text
Использование: run [опции]

Запуск zapret на переднем плане (полезно для тестов).

Опции:
   -c, --config ФАЙЛ       Загрузить конфигурацию из файла (например, conf.env)
   -s, --strategy ИМЯ      Использовать конкретную стратегию (например, discord.bat)
   -i, --interface ИМЯ     Сетевой интерфейс (по умолчанию: any)
   -t, --gamefiltertcp     Включить игровой TCP фильтр
   -u, --gamefilterudp     Включить игровой UDP фильтр
   -d, --cache-dir ПУТЬ    Папка для кэша загрузок и конфигураций
   -h, --help              Показать эту справку
```

### Примеры запуска:

*   **Запуск с определенной стратегией на интерфейсе eth0:**
    ```bash
    sudo ./zapret-rust -s discord.bat -i eth0
    ```
*   **Запуск с включенными игровыми фильтрами:**
    ```bash
    sudo ./zapret-rust -s general.bat -t -u
    ```
*   **Запуск с использованием готового конфигурационного файла:**
    ```bash
    sudo ./zapret-rust --config conf.env
    ```

## 📂 Структура конфигурации и кэша

Все скачанные зависимости, конфигурации и пользовательские списки хранятся в рабочей директории кэша (по умолчанию рядом с исполняемым файлом, либо задается через аргумент `--cache-dir` / переменную окружения `ZAPRET_CACHE_DIR`):

*   `bin/` — исполняемые файлы `nfqws` или `winws.exe` + библиотеки WinDivert.
*   `zapret-discord-youtube-linux/` — склонированный репозиторий со стратегиями обхода.
*   `conf.env` — файл настроек последнего сеанса TUI. Формат файла:
    ```env
    interface=any
    strategy=discord.bat
    gamefiltertcp=true
    gamefilterudp=true
    # Параметр только для линукса:
    backend=nftables
    ```

## 🛠️ Управление фоновой службой (Автозапуск)

Вы можете настроить запуск `zapret-rust` в фоне в качестве системной службы.

### Как это сделать:
1. Запустите TUI режим.
2. Выберите сетевой интерфейс, подходящую стратегию обхода и игровые фильтры.
3. Перейдите в раздел **⚙️ Служба (Фон)** (Service Management).
4. Выберите **⬇️ Установить службу** (Install Service).
5. Запустите службу через пункт **▶️ Запустить службу** (Start Service).

> [!IMPORTANT]
> *   На **Linux** служба регистрируется под именем `zapret-rust.service` (или аналогичным в зависимости от вашей Init-системы) и настраивается на автоматический запуск при старте системы (`WantedBy=multi-user.target`).
> *   На **Windows** служба устанавливается как системная служба Windows с флагом запуска `--service`.

## 🛡️ Исключения Windows Defender

Если встроенный антивирус Windows блокирует запуск `winws.exe` или драйвера `WinDivert`, вы можете внести рабочую папку программы в исключения прямо из TUI:
1. Запустите TUI под Администратором.
2. Откройте **🛡️ Исключения Defender** (Defender Exclusions).
3. Нажмите **➕ Добавить папку в исключения** (Add folder to exclusions).
4. Запустится фоновый процесс PowerShell, который безопасно добавит рабочую директорию в белый список антивируса.

## 🔍 Автотюн: проверки блокировок

Автотюн определяет **каким именно способом** ваш провайдер блокирует сайты, а затем **тестирует стратегии обхода** на реальных доменах. Весь процесс проходит в несколько этапов.

### Вкладки основного меню

| Вкладка | Описание |
|:---|:---|
| **🌐 Домены** | Выбор набора доменов для проверки. Доступные пресеты: Discord, YouTube, Social (Twitter, Reddit, Telegram и др.),Custom(домены пользователя). Можно выбрать несколько пресетов одновременно. |
| **📊 Запросов/домен** | Количество curl-запросов на каждый протокол для каждого домена. Если хотя бы один из запросов упал — протокол считается заблокированным. По умолчанию 3. |
| **📋 Стратегии** | Выбор стратегий обхода для тестирования. Автотюн запустит каждую стратегию, подождёт 3 секунды, а затем проверит, можно ли достучаться до заблокированных доменов через curl. |
| **🔌 Протоколы** | Какие протоколы проверять для каждого домена: HTTP, HTTPS, TLS 1.2, TLS 1.3, QUIC. |
| **🚫 Блокировки** | Какие методы DPI-блокировки проверять на уровне сети (DNS-спуфинг, TCP RST, SNI, SIBERIAN, QUIC, CIDR). |
| **✏️ Список кастомных доменов** | Открыть файл со списком доменов для редактирования. Используется при выборе пресета Custom. |
| **📊 Результаты** | Просмотр результатов последнего запуска автотюна. |
| **▶️ Запустить автотюн** | Запуск диагностики. Автоматически выполнит все выбранные проверки и тесты. |
| **🔙 Назад** | Возврат в главное меню. |

> [!IMPORTANT]
> ⚠️ **Отключите все обходы блокировок перед запуском автотюна!** Если zapret/nfqws уже запущен — автотюн не стартует с ошибкой.

### Вкладка «🔌 Протоколы»

| Протокол | Что проверяет |
|:---|:---|
| **HTTP (порт 80)** | curl-запрос по HTTP. Большинство сайтов перенаправляют на HTTPS (3xx), это считается успехом. |
| **HTTPS (порт 443)** | curl-запрос по HTTPS с TLS 1.3. Основной способ проверки доступности. |
| **TLS 1.2** | curl-запрос с принудительным TLS 1.2. Проверяет, работает ли устаревшая версия TLS. |
| **TLS 1.3** | curl-запрос с принудительным TLS 1.3. Проверяет, работает ли современная версия TLS. |
| **QUIC (UDP 443)** | curl-запрос через QUIC (HTTP/3, UDP). |

### Вкладка «🚫 Блокировки»

| Проверка | Описание |
|:---|:---|
| **DNS-спуфинг** | Сравнивает IP-адреса доменов с эталонными. Если не совпадают — DNS подменяется. |
| **TCP RST** | Устанавливает TCP-соединение и проверяет, сбрасывает ли DPI пакеты (RST) или таймаутит. |
| **SNI** | Сравнивает подключение по доменному имени vs прямому IP. Если IP работает, а домен нет — DPI блокирует по имени сервера в TLS. |
| **SIBERIAN** | Открывает 15+ параллельных соединений к одному IP. Если часть не устанавливается — провайдер ограничивает количество соединений. |
| **QUIC** | Отправляет UDP-заголовки QUIC на IP-адреса и проверяет ответ. Если нет — QUIC-трафик заблокирован. |
| **CIDR** | Тестирует подключения к IP в разных подсетях (Cloudflare, Google DNS, Discord CDN). Если часть заблокирована — блокировка по IP-диапазонам. |

### Интерпретация результатов

**Сетевые проверки:**
- ✅ = Метод блокировки не обнаружен
- ❌ = Метод блокировки обнаружен (выбирайте стратегию с обходом этого метода)
- ⏭️ = Проверка не выполнена (проблема с интернетом)

**Проверки протоколов доменов (до запуска стратегии):**
- ✅ = Протокол работает (доступен без обхода)
- ❌ = Протокол заблокирован
- ⏭️ = Пропущено (домен недоступен)

**Проверки стратегий:**
- ✅ = Стратегия помогает (домены доступны)
- ❌ = Стратегия не помогает (домены заблокированы)
- Стратегия считается «работающей», если помогает **больше половины** заблокированных доменов.
